DESIGN GOALS
================================================================================
1. Dropbox is source of truth — chat is not the record.
2. Hybrid autonomy: auto-write simple logs; owner approves money/tax/job-status.
3. Multi-source knowledge — tiered context loader.
4. Multi-provider — OpenAI first; Anthropic, Cursor SDK, Gemini, Ollama later.
5. Easy to work on — app/office_ai/ package, one file per tool/provider.
6. Security first — admin-only Phase 1; API keys never in Dropbox logs.

ROUTES (PHASE 1)
================================================================================
  /office-ai              Main chat (admin only)
  /office-ai/approvals    Pending write actions with diff preview
  /ai                     Provider keys, sources, test connection
  /api/office-ai/chat     POST message JSON API
  /api/office-ai/approve/<id>  Approve or deny pending action

Distinct from Live Chat (/chat) — team messaging, not LLM.

================================================================================
HYBRID AUTONOMY
================================================================================
AUTO (no approval):
  - Append daily log note
  - Save receipt path note to job folder
  - Read dashboard, job register, job folder, chatgpt imports search

REQUIRES OWNER APPROVAL:
  - Financial CSV changes (payroll, income, tax, helper overhead)
  - Full workspace sync
  - PDF generation / send drafts
  - User/role changes
