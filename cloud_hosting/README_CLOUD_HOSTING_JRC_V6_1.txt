J & R Construction Manager v6.3 - Live Cloud Hosting

Recommended live options:
- Render/Railway/Fly.io/Docker VPS: best for always-on public customer/worker access.
- Cloudflare Tunnel: good when publishing a local/VPS service without opening inbound firewall/router ports.

Required production variables:
JRC_PUBLIC_HOST_MODE=1
JRC_SECRET_KEY=<long random secret generated by the host or password manager>
JRC_TRUSTED_HOSTS=<your-domain.example.com>
JRC_CLOUD_BASE_URL=https://<your-domain.example.com>
JRC_DEVICE_COOKIE_SAMESITE=Lax
JRC_SESSION_TIMEOUT_MINUTES=120

Production command pattern:
gunicorn cloud_hosting.cloud_entry:app --workers 2 --threads 4 --timeout 90 --bind 0.0.0.0:$PORT

Test URLs after deploy:
/api/health
/api/connection
/api/cloud/status
/connect
/login
/mobile
/register
/apply
/customer/request
/live-deployment-readiness

Security rules:
- Change admin/admin before sharing.
- Use HTTPS only for public users.
- Remembered devices require user checkbox and expire after 90 days.
- Customers see only their own portal/requests/customer-visible notes.
- Non-company users see only minimal shared items.
- Dropbox/file sources are evidence/document sources, not a live multi-user database.
